# P3: Development

**Purpose:** Implement the plan. Passing tests, no surprises.

## Rules

- Follow plan order. Diverge → update plan first.
- Stage changes, do NOT commit. All commits in P6.
- Tests/lint/build must pass before P4.
- Technical direction change → update Design Doc, flag for ADR update in P5.

## No test infrastructure

- Full-track: set up minimal test infra as first task (framework + smoke test).
- Fast-track: verify manually, note what was checked.

## Scope check

Work exceeding plan? Surface it: `[6PHASE: SCOPE CHECK | planned: N | actual: M | risk: ...]`
Options: expand, cut & defer, escalate track.

## Blockers

Update plan with learnings. Design wrong → `[6PHASE: P3 → P1|P2 | reason: ...]`. Keep work done.

## Output before P4

Implementation summary, test results, manual test instructions if applicable.

Use available framework/language/design-system skills for implementation.
