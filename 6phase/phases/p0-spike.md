# P0: Spike

**Purpose:** Answer one specific feasibility question with throwaway code.
**Output:** Findings summary (inline or `designs/YYYY-MM-DD-<slug>-spike.md`)

## When to use

Unknown feasibility, new tech, performance questions needing measurement, integration uncertainty.

## Rules

- One spike = one question. Define it before coding.
- Time-box: minutes to an hour. Too long → question too broad, split it.
- Throwaway code — no tests, no clean structure needed.
- Do NOT carry spike code into P3. Start fresh.

## Output

1. Question → 2. What you tried → 3. What you learned → 4. Recommendation: proceed (→ P1), pivot, or abandon.

Delete spike code after — don't commit it.

Use available domain-specific skills to answer faster.
